# Frontend Guidelines

Use this file for frontend-specific rules. Keep frontend details here instead of the root file whenever possible.

## Scope
- Next.js `15.4.8`, React `18.3.1`, TypeScript `5`, CopilotKit `1.53.0`, Tailwind CSS `4`
- App Router code currently lives in `frontend/src/app`
- Frontend tests should use Vitest for unit/integration and Playwright for E2E

## Frontend Conventions
- Use 2-space indentation.
- Use `PascalCase` for components and camelCase for functions and variables.
- Keep formatting and linting rules aligned with Prettier and ESLint as this area evolves.
- Add frontend-specific guidance here as it is learned instead of growing the root `AGENTS.md`.
